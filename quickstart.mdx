This guide walks you through running a Google-provided Dataflow Template using the `gcloud` CLI. You will run the **Pub/Sub to Cloud Storage Text** template, which reads messages from a Pub/Sub topic and writes them as text files in Cloud Storage.

## Prerequisites

<Steps>
  <Step title="Install the gcloud CLI">
    Download and install the [Google Cloud CLI](https://cloud.google.com/sdk/docs/install) if you have not already done so.

    Verify installation:

    ```bash
    gcloud version
    ```
  </Step>

  <Step title="Authenticate">
    Log in with your Google account and set up application default credentials:

    ```bash
    gcloud auth login
    gcloud auth application-default login
    ```
  </Step>

  <Step title="Set your project">
    Set the GCP project you want to use:

    ```bash
    gcloud config set project YOUR_PROJECT_ID
    ```

    Enable the Dataflow API if it is not already enabled:

    ```bash
    gcloud services enable dataflow.googleapis.com
    ```
  </Step>

  <Step title="Create required resources">
    Create a Cloud Storage bucket and a Pub/Sub topic for this example:

    ```bash
    # Create a Cloud Storage bucket
    gsutil mb gs://YOUR_BUCKET_NAME

    # Create a Pub/Sub topic
    gcloud pubsub topics create YOUR_TOPIC_NAME
    ```

    <Note>
      Replace `YOUR_BUCKET_NAME` and `YOUR_TOPIC_NAME` with names that are unique within your project.
    </Note>
  </Step>
</Steps>

## Run a classic template

Classic Templates (v1) are staged as files in Cloud Storage. Google provides pre-staged versions you can run directly.

<Steps>
  <Step title="Set environment variables">
    ```bash
    export PROJECT=YOUR_PROJECT_ID
    export BUCKET_NAME=YOUR_BUCKET_NAME
    export REGION=us-central1
    export INPUT_TOPIC=projects/$PROJECT/topics/YOUR_TOPIC_NAME
    export OUTPUT_DIRECTORY=gs://$BUCKET_NAME/output/
    ```
  </Step>

  <Step title="Run the template with gcloud">
    The `gcloud dataflow jobs run` command launches a classic template job:

    ```bash
    gcloud dataflow jobs run "pubsub-to-gcs-text-job" \
      --project "$PROJECT" \
      --region "$REGION" \
      --gcs-location "gs://dataflow-templates-$REGION/latest/Cloud_PubSub_to_GCS_Text" \
      --parameters "inputTopic=$INPUT_TOPIC" \
      --parameters "outputDirectory=$OUTPUT_DIRECTORY" \
      --parameters "outputFilenamePrefix=output-" \
      --parameters "outputFilenameSuffix=.txt"
    ```

    The `--gcs-location` flag points to the Google-provided template path. You can also use a template you have staged yourself.
  </Step>
</Steps>

<Tip>
  Google pre-stages all provided templates at `gs://dataflow-templates-REGION/latest/TEMPLATE_NAME`. You do not need to build or stage anything to use them.
</Tip>

## Run a flex template

Flex Templates (v2) are packaged as Docker containers. The `gcloud dataflow flex-template run` command is used instead of `gcloud dataflow jobs run`.

<Steps>
  <Step title="Set environment variables">
    ```bash
    export PROJECT=YOUR_PROJECT_ID
    export BUCKET_NAME=YOUR_BUCKET_NAME
    export REGION=us-central1
    export INPUT_TOPIC=projects/$PROJECT/topics/YOUR_TOPIC_NAME
    export OUTPUT_DIRECTORY=gs://$BUCKET_NAME/output/
    export TEMPLATE_SPEC_GCSPATH="gs://dataflow-templates-$REGION/latest/flex/Cloud_PubSub_to_GCS_Text_Flex"
    ```
  </Step>

  <Step title="Run the flex template">
    ```bash
    gcloud dataflow flex-template run "pubsub-to-gcs-flex-job" \
      --project "$PROJECT" \
      --region "$REGION" \
      --template-file-gcs-location "$TEMPLATE_SPEC_GCSPATH" \
      --parameters "inputSubscription=projects/$PROJECT/subscriptions/YOUR_SUBSCRIPTION" \
      --parameters "outputDirectory=$OUTPUT_DIRECTORY" \
      --parameters "outputFilenamePrefix=output-" \
      --parameters "outputFilenameSuffix=.txt"
    ```

    <Note>
      Flex templates use `--template-file-gcs-location` instead of `--gcs-location`, and the path includes the `flex/` prefix.
    </Note>
  </Step>
</Steps>

## Run via Google Cloud Console

You can also launch templates through the browser without using the CLI.

<Steps>
  <Step title="Open the Dataflow console">
    Go to the [Dataflow console](https://console.cloud.google.com/dataflow) in your GCP project.
  </Step>

  <Step title="Create a job from template">
    Click **Create job from template**. In the **Dataflow template** dropdown, search for and select **Pub/Sub Subscription to Text Files on Cloud Storage**.
  </Step>

  <Step title="Fill in the parameters">
    Provide values for the required parameters:

    - **Input Pub/Sub subscription**: your subscription name
    - **Output file directory in Cloud Storage**: your bucket path (e.g., `gs://YOUR_BUCKET_NAME/output/`)
    - **Output filename prefix**: `output-`

    Select a **Region** and click **Run job**.
  </Step>
</Steps>

## Monitor the job

<Tabs>
  <Tab title="gcloud CLI">
    List your running Dataflow jobs:

    ```bash
    gcloud dataflow jobs list --region us-central1
    ```

    Describe a specific job:

    ```bash
    gcloud dataflow jobs describe JOB_ID --region us-central1
    ```

    Stream job logs:

    ```bash
    gcloud dataflow jobs show JOB_ID --region us-central1
    ```
  </Tab>
  <Tab title="Google Cloud Console">
    Open the [Dataflow Jobs page](https://console.cloud.google.com/dataflow/jobs). Click your job to view:

    - **Job graph**: the pipeline stages and data flow
    - **Logs**: worker and job-level log output
    - **Metrics**: throughput, element counts, and system lag (for streaming jobs)
  </Tab>
</Tabs>

<Info>
  For streaming templates like Pub/Sub to Cloud Storage, the job runs continuously until you cancel it. Cancel a running job with:

  ```bash
  gcloud dataflow jobs cancel JOB_ID --region us-central1
  ```
</Info>
